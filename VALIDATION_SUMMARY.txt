================================================================================
  KNOWN ISSUES & ROOT CAUSES
================================================================================

ISSUE #1: 0-Star Images (4 failures)
  ├─ Symptom: Returns None instead of 0
  ├─ Root Cause: Algorithm can't detect "empty rating grid"
  ├─ Examples: 
  │  - "No color star at all" → None (should be 0)
  │  - "No stars in this picture" → None (should be 0)
  └─ Fix Needed: Implement empty grid detection using Otsu+Canny edge detection
     Impact: +6-7% accuracy (to ~72%)

ISSUE #2: 5-Star Undercounting (6 failures → detected as 4★)
  ├─ Symptom: "5 Green star in a row" detected as 4
  ├─ Root Cause: Morphological MORPH_OPEN with (3,3) kernel merges adjacent blobs
  ├─ Examples:
  │  - Row 10, 16, 17, 40, 49, 58: All "5 Green stars" → 4 detected
  └─ Fix Needed: Reduce kernel from (3,3) to (2,2) or increase max_area threshold
     Impact: +10% accuracy (to ~75%)

ISSUE #3: Mixed-Color Arrangements (2 failures)
  ├─ Symptom: "1 Gold + 4 dark stars" detected as slot 1 or 3
  ├─ Root Cause: Slot detection only sees the gold blob, position is ambiguous
  ├─ Example:
  │  - Row 33, 41: Mixed colors → wrong slot position
  └─ Fix Needed: Add heuristic for mixed arrangements (sum visible blobs)
     Impact: +2-3% accuracy

ISSUE #4: None Detection on Green Stars (3 failures)
  ├─ Symptom: Some green star images return None instead of count
  ├─ Root Cause: HSV mask not capturing all green shades
  ├─ Example:
  │  - Rows 18, 34, 46: "5 Green colored star" → None (fallback to OCR)
  └─ Fix Needed: Expand green HSV ranges [35-100 hue, 20-255 sat, 30-220 val]
     Impact: +2-3% accuracy (to ~68%)
